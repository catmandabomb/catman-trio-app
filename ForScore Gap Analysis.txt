FORSCORE vs CATMAN TRIO APP — FEATURE GAP ANALYSIS
=====================================================
Generated: March 2026

What ForScore has that Catman Trio doesn't, ranked by usefulness
for a live performing band. Focused on Live Mode / performance.

=====================================================================
TIER 1: HIGH IMPACT FOR LIVE PERFORMANCE
=====================================================================

1. SCREEN WAKE LOCK
   Effort: Trivial (5 lines)
   ForScore keeps the screen on automatically. Without this, the
   screen goes dark mid-song — a showstopper at a gig.
   Uses navigator.wakeLock API.
   STATUS: ADDED in v17.83 (Live Mode only)

2. HALF-PAGE TURNS
   Effort: Medium
   ForScore's signature feature. Instead of a full page flip, shows
   the bottom half of the current page and top half of the next page,
   so you can read ahead before the turn. Eliminates the "panic flip"
   moment where you can't see what's coming.

3. PAGE-TURN LINK ZONES (Repeat/DS/Coda Jumps)
   Effort: Medium-High
   Tap a region on the chart to jump to a different page. Essential
   for charts with repeats, DS al Coda, or navigation marks. Without
   this, musicians must manually swipe back and forth during the song.
   Implementation: Define tap zones on the PDF canvas that map to
   target page numbers. Could be set up in edit mode.

4. CHART ANNOTATIONS
   Effort: High
   Freehand drawing, text notes, and highlights directly on the PDF
   canvas. Musicians mark dynamics, cues, arrangement changes during
   rehearsal. These persist per-song. Currently charts are read-only
   canvases. Would need a canvas overlay layer + storage for
   annotation data (could use GitHub data branch).

5. MIDI PROGRAM CHANGE OUTPUT
   Effort: Medium
   When the song changes in Live Mode, send a MIDI program change
   message. This lets guitarists/keyboardists auto-switch presets on
   their pedalboards or synths. The Web MIDI API is available in
   Chromium browsers. Would need per-song MIDI config (channel +
   program number) in the song metadata.

6. PER-PAGE MARGIN CROPPING
   Effort: Medium
   Crop/adjust margins on each PDF page so the actual music fills
   more of the screen. Many charts have huge headers, footers, or
   margins that waste precious screen space. ForScore lets you set
   crop per-page. Implementation: store crop rect per page in song
   metadata, apply as viewport offset during rendering.


=====================================================================
TIER 2: VERY USEFUL FOR REHEARSAL AND PREP
=====================================================================

7. BUILT-IN METRONOME
   Effort: Medium
   The app stores BPM per song but has no metronome. A tap-to-start
   metronome that reads the song's BPM (with tap-tempo override)
   would be valuable for rehearsal and soundcheck count-ins. Could
   use Web Audio API for precise timing.

8. AUTO-SCROLL
   Effort: Medium
   For long single-page charts or continuous scroll view, auto-scroll
   at an adjustable speed so the musician doesn't need to touch the
   device. Speed could be derived from BPM + page length. Useful for
   long lead sheets.

9. TWO-PAGE / LANDSCAPE DISPLAY
   Effort: Medium
   Show two pages side-by-side on a tablet in landscape orientation.
   Many musicians use iPads in landscape specifically for this.
   Detection via screen.orientation or matchMedia query.

10. CUSTOMIZABLE PEDAL MAPPING
    Effort: Medium
    Currently pedals only do forward/back. ForScore lets you map pedal
    buttons to any action (e.g., left pedal = previous song's first
    page, right pedal = next page, center pedal = toggle metronome).
    Would need a pedal config screen.

11. DARK/INVERTED SCORE MODE
    Effort: Low
    Invert the PDF colors (white notes on black background) for dark
    stage environments. Reduces screen glare and eye strain on dimly
    lit stages. Could use CSS filter: invert(1) on the canvas, or
    a canvas compositing mode. Toggle button in Live Mode header.


=====================================================================
TIER 3: NICE TO HAVE
=====================================================================

12. AUDIO A-B LOOPING WITH SPEED CONTROL
    The app has an audio player, but ForScore lets you slow down
    playback and loop a section for practice. Uses Web Audio API
    playbackRate property. Useful for learning tricky passages.

13. MIDI PROGRAM CHANGE INPUT
    Load a specific setlist or jump to a song via incoming MIDI.
    Useful when the bandleader's device controls everyone's.

14. SMART SETLISTS (Auto-Populate by Rules)
    Create setlists that auto-populate based on metadata filters
    (e.g., "all songs tagged 'jazz' in key of Bb"). The app has tags
    and filtering in the song list but not auto-generated setlists.

15. EXTERNAL DISPLAY / AIRPLAY
    Show the chart on a separate monitor via the Presentation API
    while keeping controls on the primary device. Useful for music
    stands with external monitors.

16. SETLIST EXPORT (Text/CSV)
    Export the setlist as a plain text list for printing or sharing
    with sound engineers. The app has "Copy" but a formatted export
    would be more useful.

17. ANNOTATION LAYERS (Show/Hide)
    Toggle annotations on/off. Useful when sharing the same chart
    between rehearsal (marked up) and performance (clean).

18. PRACTICE TIME TRACKING
    Log how long you spend on each song. ForScore tracks this
    automatically. Could tie into the existing practice lists feature.


=====================================================================
WHAT CATMAN TRIO ALREADY HAS (that ForScore also has)
=====================================================================

- Setlists with song ordering and drag reorder
- Live Mode with fullscreen, carousel swipe, slide transitions
- PDF chart rendering via PDF.js on canvas
- Pinch-to-zoom, wheel zoom, double-tap zoom, drag-to-pan
- Keyboard/pedal navigation (Arrow keys, PageDown, Space, PageUp)
- Quick-jump song picker (jump to any song in setlist)
- Song overlay on song boundaries (title, key, BPM, time sig)
- Session restore (remembers position if you exit accidentally)
- Progress indicator (Song X/Y, Page X/Y)
- Clock and set timer in Live Mode header
- Metadata slides for songs without charts
- Multi-chart support per song with ordered chart selection
- Haptic feedback on navigation events
- Fullscreen API integration
- Offline PDF caching via service worker
- Screen Wake Lock in Live Mode


=====================================================================
RECOMMENDED "DO THESE NEXT" ORDER
=====================================================================

Priority 1: Half-Page Turns (ForScore's #1 differentiator)
Priority 2: Dark/Inverted Score Mode (low effort, high stage value)
Priority 3: Built-in Metronome (natural fit with existing BPM data)
Priority 4: Link Zones for repeats/DS/Coda
Priority 5: MIDI Program Change Output
