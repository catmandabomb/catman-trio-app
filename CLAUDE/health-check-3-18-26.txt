6. CLASSIC 4 FIX STATUS
=========================================
Status: 4 FIXES APPLIED, AWAITING iOS TESTING

Fix 1 — pdf-viewer.js: retry on containerWidth <= 0
  Up to 5 retries using rAF + setTimeout(50ms).
  Prevents silent abort when container hasn't been laid out yet.

Fix 2 — setlists.js: carousel starts with opacity:1
  iOS was skipping canvas compositing when opacity was 0.

Fix 3 — setlists.js: _afterSnap awaits render
  Previously reset carousel position (translateX) before PDF render finished,
  causing a flash-then-black effect. Now awaits the render promise.

Fix 4 — setlists.js: wait-for-load polling (UNCOMMITTED)
  When _renderPageIntoSlide is called but pdfDoc is still null (PDF still
  downloading), instead of returning Promise.resolve() immediately, it now
  polls every 200ms for up to 12s. When the PDF loads, it re-invokes
  _renderPageIntoSlide to actually render the page.
  This was the deepest root cause — carousel rendered "loading" state
  permanently because the render call happened before the PDF existed.

All 4 fixes are complementary:
  Fix 1 = container ready, Fix 2 = canvas visible
  Fix 3 = timing correct, Fix 4 = PDF available
