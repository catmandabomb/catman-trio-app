## Piano Playback Expansion Research (for 25-year pianist)

### Voice Leading â€” Algorithm Upgrade
**Current:** Simple closest-inversion heuristic (minimize distance).
**Proposed scoring function:**
```
score = distancePenalty + (parallelPenalty * 2) + functionBonus
```
- **Parallel 5ths/octaves check**: Enforce >2-semitone separation on voice jumps
- **Soprano constraint**: Keep top voice smooth (<6 semitone jumps)
- **Functional harmony**: Root position on V chords, first-inversion on passing chords
- **Bass-soprano spread**: Keep under 24 semitones

### Voicing Types to Implement (priority order)
| Type | Description | Priority |
|------|-------------|----------|
| **Shell** | Root-3-7 only | HIGH â€” implement first |
| **Drop-2** | Closed voicing with 2nd-from-top dropped an octave | HIGH |
| **Drop-3** | Orchestral texture | MEDIUM |
| **Rootless** | 3-7-9-13 (no root) | MEDIUM â€” modern jazz |
| **Tensions** | 9/11/13 on Maj7, min7 | MEDIUM |

### Sound Quality Options (ranked)
| Approach | Fidelity | Size | Notes |
|----------|----------|------|-------|
| **SoundFont2** | Near-acoustic | ~16KB | Best. Load via ESM, store on R2. |
| **Karplus-Strong** | Piano-convincing | ~3KB code | Plucked string model with damping. Good fallback. |
| **FM synthesis** | Warmer than oscillator | ~2KB | Chowning method. |
| **Current oscillators** | Thin/synthy | 0 extra | Keep as final offline fallback. |

### Comping Patterns per Style
| Style | Pattern |
|-------|---------|
| **Swing** | Shell on 1,3 â€” tensions on &-of-2, &-of-4 |
| **Bossa** | Root-position on 1, drop-2 on syncopation (clave-aware) |
| **Ballad** | Whole/half notes, light arpeggios |
| **Funk** | Sharp stabs on 1,&,3,& â€” quick attack/decay |
| **Blues** | Dominant shapes, groove-locked 16ths |

### Implementation Phases
1. **Phase 1** (next session): Shell voicings + parallel-5th checking + scoring algorithm
2. **Phase 2** (following session): SoundFont2 via ESM + Karplus-Strong fallback
3. **Phase 3** (polish): Comping pattern modes with style-keyed rhythm maps

Entry point: `_buildVoicing()` in `js/wikicharts.js`

### AudioWorklet Metronome Upgrade

From Cutting Edge Research â€” this is the **single most impactful audio upgrade**:

Our current metronome uses the classic lookahead pattern (setTimeout + OscillatorNode.start). It works but is vulnerable to main-thread jank (tab switches, heavy DOM updates, etc.).

**AudioWorklet metronome** runs on the audio thread:
- `process()` called every 128 samples (~2.9ms at 44.1kHz)
- Maintains sample counter, generates click at beat boundaries
- **Immune to main thread jank** â€” no drift, no stuttering
- Posts `{ type: 'beat', beat: n }` to main thread for UI flash
- Baseline since April 2021, all browsers including iOS Safari 14.5+

**Effort: 3-4 hours.** The backing track scheduler (`BackingTrackScheduler`) should also be built on AudioWorklet from the start.

### OffscreenCanvas PDF Rendering

From Cutting Edge Research â€” called "THE SINGLE MOST IMPACTFUL UPGRADE AVAILABLE" for PDF:

Move PDF.js rendering to a Web Worker via OffscreenCanvas:
1. Main thread creates `<canvas>`, calls `canvas.transferControlToOffscreen()`
2. Transfers to Worker via `postMessage`
3. Worker runs PDF.js rendering (CPU-heavy) without blocking UI
4. Result appears on screen with zero main-thread cost

**Baseline since March 2023.** All browsers including iOS Safari 16.4+.
**Effort: 4-6 hours.** Eliminates the #1 cause of jank during page turns in Live Mode.
