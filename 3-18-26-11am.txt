2. SET UP EMAIL_FROM_PERSONAL VARIABLE
=============================================

The email setlist feature (v19.96+) uses a "from" address when sending
setlist emails via Resend. There are two env vars:

  - EMAIL_FROM          = used for all system emails (login alerts,
                          password reset, verification, lockout alerts)
  - EMAIL_FROM_PERSONAL = used ONLY for the "email setlist" feature
                          (line 992 of index.js). Falls back to
                          EMAIL_FROM if not set.

This lets you send setlist emails from your personal address (e.g.
"Cat <cat@catmanbeats.com>") while system emails come from a
no-reply address.

STEPS:

  a) Set EMAIL_FROM (if not already set):

     wrangler secret put EMAIL_FROM
     > Enter value: Catman Trio <noreply@catmanbeats.com>

  b) Set EMAIL_FROM_PERSONAL:

     wrangler secret put EMAIL_FROM_PERSONAL
     > Enter value: Cat <cat@catmanbeats.com>

     (Use whatever personal "from" address you want recipients to see
      when they get emailed setlists.)

  NOTE: Both of these require a verified domain in Resend. If you're
  using catmanbeats.com, make sure it's verified in your Resend dashboard
  at https://resend.com/domains. The fallback "onboarding@resend.dev"
  works without verification but looks generic.

  c) After setting secrets, redeploy:

     wrangler deploy
